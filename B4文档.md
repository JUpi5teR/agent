# B4 模块文档：本地 Agent LLM 调用

## 概述

**入口文件**：`code/b4_local_agent_llm.py`

**核心职责**：使用 mock 模式或本地 LLM 生成标准 `AIMessage`，**不执行工具**。

B4 是 Agent 架构中的 LLM 调用层，负责接收对话历史和工具 schema，返回模型生成的 `AIMessage`（包含 `content` 和/或 `tool_calls`）。工具的实际执行由 B3 负责。

---

## 两种运行模式

### 1. Mock 模式（`--mode mock`）

不加载真实模型，用于：
- 无 GPU 环境下的调试
- 无本地模型时的模块联调
- 快速验证流程

Mock 模式的行为：
- 如果没有 `tool` 消息：返回一个预设的 `file_reader` 工具调用
- 如果有成功的 `tool` 消息：从工具结果中提取内容，生成三条中文要点作为最终回答
- 如果工具执行失败：返回说明失败原因的错误回答

### 2. Prompt JSON 模式（`--mode prompt_json`）

加载本地模型（通过 `transformers` 库），真实运行推理：
- 使用 `model.yaml` 中的配置加载本地 Qwen3.5-4B 模型
- 支持 `bf16`/`fp16`/`fp32` 等精度配置
- 支持 `device_map="auto"` 自动分配设备

## 核心函数

### `generate_ai_message(model_config, messages, tools_schema, mode, artifact_dir, artifact_stem)`

**输入参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `model_config` | `str \| Path` | 指向 `model.yaml` 配置文件 |
| `messages` | `list[dict]` | 对话历史，包含 `system`、`user`、`tool` 等角色 |
| `tools_schema` | `list[dict]` | 可用工具的 JSON Schema 描述 |
| `mode` | `str` | `"mock"` 或 `"prompt_json"` |
| `artifact_dir` | `str \| None` | 输出目录 |
| `artifact_stem` | `str \| None` | 输出文件名前缀 |

**返回**：
```python
{
    "ai_message": {  # 标准化的 AIMessage
        "role": "assistant",
        "content": "最终回答文本",
        "tool_calls": [...]
    },
    "status": "success" | "error",
    "error": None | {"type": "...", "message": "..."}
}
```

---

## 输出格式

### AIMessage 标准结构

```python
{
    "role": "assistant",
    "content": "最终回答文本",      # 二选一：content 非空则 tool_calls 必须为空
    "tool_calls": [                # 二选一：tool_calls 非空则 content 必须为空
        {
            "id": "call_001",
            "name": "tool_name",
            "args": {"param": "value"}
        }
    ]
}
```

**重要约束**：
- `content` 和 `tool_calls` **不能同时存在**，必须二选一
- 这是 B4 的核心契约，保证与 B3 的清晰边界

---

## 内部处理流程

### 1. Prompt 构建（`_build_prompt_messages`）

将对话历史转换为模型输入格式，注入：
- 系统级别的工具 schema 描述
- 严格的输出格式指令（强制 JSON 输出，不得包含 markdown）
- 两条 schema 约束：
  - Schema A：`{"content":"final answer","tool_calls":[]}`
  - Schema B：`{"content":"","tool_calls":[{"id":"...","name":"...","args":{}}]}`

### 2. 模型调用（`_prompt_json_generate`）

使用 `transformers` 库的典型流程：
```python
inputs = tokenizer.apply_chat_template(...)
with torch.no_grad():
    generated = model.generate(**inputs, ...)
output = tokenizer.decode(new_tokens, ...)
```

### 3. 输出解析（`_parse_model_output`）

支持三种解析策略，依次尝试：

1. **直接 JSON 解析**：尝试 `json.loads(raw_text.strip())`
2. **尾部反引号过滤**：处理 `` ` `` `` 包裹的 JSON
3. **工具调用片段提取**：从 `"tool_calls":[...` 标记处截取数组

解析后的验证：
- 检查 `content` 和 `tool_calls` 的互斥性
- 规范化 `tool_calls` 的格式

### 4. 错误处理

如果所有解析策略均失败：
- `status` 设为 `"error"`
- `ai_message` 设为包含解析错误提示的默认消息
- `PARSE_ERROR_CONTENT` = "模型输出解析失败，无法生成有效工具调用或最终回答。"

---

## 模型缓存

B4 实现了简单的内存缓存（`_MODEL_CACHE`），避免重复加载模型：

```python
cache_key = (model_path, tokenizer_path, local_only, trust_remote_code, dtype, device_map_key, max_memory_key)
```

重复调用时直接返回缓存的 `(tokenizer, model)`。

---

## 输出文件

当指定 `--outdir` 时，B4 生成以下文件：

| 文件 | 内容 |
|------|------|
| `<stem>raw_model_output.json` | 原始生成文本、解析状态、backend、错误信息 |
| `<stem>ai_message.json` | 规范化后的 AIMessage |
| `llm_run_log.jsonl` | 运行历史日志（追加模式） |

---

## 与其他模块的关系

```
B1 (Agent Runtime)
    └── B4 (LLM 调用) ──生成 AIMessage ─→ B3 (工具执行)
                                                │
    B5 (Memory) ←────────────────────────────────┘
```

- **B1**：总控调度，决定何时调用 B4
- **B3**：接收 B4 生成的 `tool_calls`，执行工具，返回 `ToolMessage`
- **B4**：只负责生成 `AIMessage`，不执行任何工具
- **B5**：根据对话内容管理记忆

---

## 典型使用场景

### 场景 1：生成工具调用

输入：`messages` 只有 `system` + `user`，无工具结果
预期输出：AIMessage 包含 `tool_calls`

### 场景 2：生成最终回答

输入：`messages` 包含 `ToolMessage`（工具执行结果）
预期输出：AIMessage 的 `content` 包含最终回答，`tool_calls` 为空

### 场景 3：处理工具执行失败

输入：`messages` 包含 `status="error"` 的 `ToolMessage`
预期输出：AIMessage 的 `content` 说明失败原因，`tool_calls` 为空

---

## 配置示例（model.yaml）

```yaml
model:
  model_name_or_path: /path/to/Qwen3.5-4B
  tokenizer_name_or_path: /path/to/Qwen3.5-4B  # 可省略，默认为 model 路径
  backend: transformers
  torch_dtype: bfloat16  # bfloat16 / float16 / float32 / auto
  device_map: auto
  max_memory: null
  local_files_only: true
  trust_remote_code: false

generation:
  max_new_tokens: 1024
  do_sample: false
```
